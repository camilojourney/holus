# Customer Response Template

> Standard format for support responses

---

## Basic Structure

```
Hi [Name],

[Acknowledge their situation]

[Explain what's happening / Answer their question]

[Provide next steps or resolution]

[Optional: Offer additional help]

Best,
[Signature]
```

---

## Templates by Situation

### Bug Report Received

```
Hi [Name],

Thanks for reporting this issue! I can see how frustrating that would be.

I've logged this as a bug and we're looking into it. [If you have more context, share it.]

I'll follow up once we have a fix. In the meantime, [workaround if any].

Best,
[Name]
```

### Confused/Frustrated User

```

Let me help clarify: [Clear explanation]

To fix your situation: [Specific actionable steps]

Best,
[Name]
```

### Can't Help / Not Possible

```
Hi [Name],

Thanks for reaching out! I wish I had better news, but [explain limitation honestly].

Here's what you can do instead: [Alternative if any]

[If it's something we might build: "I've noted this as a feature request."]

Best,
[Name]
```

---

## Tone Guidelines

### Do
- Be warm but not fake
- Acknowledge their experience
- Be specific and helpful
- Offer next steps
- End with an opening for follow-up

### Don't
- Be robotic or template-y
- Make promises you can't keep
- Use jargon they won't understand
- Blame them for the issue
- Leave them hanging

---

## Escalation Triggers

Escalate to founder if:
- User is very angry
- Refund request
- Legal or compliance issue
- Public complaint (social media)
- VIP or high-value customer
- Data loss or security issue

---

## Response Time Targets

| Priority | Response Time |
|----------|---------------|
| Security/Data issue | < 1 hour |
| Bug blocking user | < 4 hours |
| General question | < 24 hours |
| Feature request | < 48 hours |
