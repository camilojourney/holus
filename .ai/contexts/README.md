# Context Files

This directory stores project context used by AI agents.

## Core Files

- `product-context.md`: product vision, users, constraints
- `current-priorities.md`: current sprint priorities and blockers

## Optional Files

- `domain-context.md.example`: domain terminology and constraints
- `holus.md`: deep project context (create when needed)

## Usage

- Keep context concise and current
- Update priorities weekly
- Load context on demand, not all at once

## Good Practices

- Use concrete, actionable language
- Link to specs/issues for details
- Remove stale priorities quickly
