# Communication Voice Standards

## Principles

- Clear, direct, and actionable
- Respectful and professional
- Specific over vague
- Short sentences over dense paragraphs

## Documentation

- Lead with outcome and prerequisites
- Use step-by-step procedures for tasks
- Include examples when behavior is non-obvious
- Keep terminology consistent across docs and UI

## UI Copy

- Prefer action verbs for buttons
- Explain errors with next-step guidance
- Keep empty states informative and actionable
- Avoid jargon unless defined nearby

## Support Responses

- Acknowledge issue succinctly
- Explain what happened in plain language
- Provide concrete next actions
- Avoid blame-oriented wording

## Style Rules

- Sentence case headings
- Present tense, active voice
- Use explicit nouns and verbs
- Avoid unnecessary exclamation points

## QA Checklist

- [ ] Terminology consistent
- [ ] Error messages actionable
- [ ] Instructions testable as written
- [ ] Tone matches product voice
